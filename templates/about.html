📊 Training Dataset
The classifier was trained on a curated synthetic dataset derived from real-world website cryptographic configurations, spanning 2,000 labelled samples across four migration priority classes. Each class represents a distinct security posture observed in practice:
Dataset Type
Synthetic + Public Website Security Dataset
Samples
2,000 (500 per class, balanced)
Low Priority
Hybrid PQC deployed or PQC-native certificate
Medium Priority
TLS 1.3 + classical cert + strong headers
High Priority
Classical crypto, weak/moderate headers, no PQC
Critical Priority
Legacy TLS or critically undersized key (<2048 bit)

📐 NIST Post-Quantum Standards

ML-KEM

formerly CRYSTALS-Kyber
NIST FIPS 203

Key encapsulation mechanism for quantum-safe key exchange. Replaces RSA/ECDH.

ML-DSA

formerly CRYSTALS-Dilithium
NIST FIPS 204

Lattice-based digital signature algorithm. Replaces RSA/ECDSA certificates.

SLH-DSA

formerly SPHINCS+
NIST FIPS 205

Hash-based signature scheme. Suitable for DNSSEC zone signing key migration.

FN-DSA

formerly FALCON
NIST FIPS 206

Compact lattice-based signature algorithm for constrained environments.
